# Role: 资深 Python 架构师 & 数据工程师

# 任务背景
我正在完成一个面试题目：**矿业数据聚合管线**。
目标是从零构建一个工程化、模块化、可测试的 Python 系统，包含 ETL 流水线、向量数据库存储和 API 服务。

# 需求概览
1. **数据采集**：从 3 个源（矿业新闻、政策、价格）采集数据，每源至少 200 条。
2. **数据处理**：清洗、去重、向量化（Embedding）。
3. **存储**：存入向量数据库（ChromaDB）。
4. **服务**：提供 FastAPI 接口 `/query` 支持自然语言检索。
5. **评估**：包含自动化评估脚本。

# 技术栈约束
- **语言**: Python 3.10+
- **Web框架**: FastAPI
- **数据库**: ChromaDB (Persistent Client)
- **向量化模型**: sentence-transformers (all-MiniLM-L6-v2)
- **爬虫**: requests, BeautifulSoup4, feedparser
- **工具**: Pydantic (数据校验), tqdm (进度条), loguru (日志)
- **规范**: PEP8, Type Hints, Google Style Docstrings

# 执行步骤

请按照以下步骤逐步执行，每完成一步请询问我是否继续：

## 第一步：项目初始化与架构设计
请作为资深架构师，设计符合工程化规范的项目目录结构。
- 要求包含：`pipeline/` (ETL逻辑), `serve/` (API服务), `eval/` (评估测试), `tests/` (单元测试), `configs/` (配置)。
- 输出目录树结构，并解释每个模块的职责。

## 第二步：核心数据模型定义
在 `pipeline/models.py` 中定义 Pydantic 模型 `MiningData`。
- 字段：`id` (UUID), `source_type` (Enum: news, policy, price), `title`, `content`, `publish_date`, `metadata` (JSON), `embedding` (List[float], Optional)。
- 要求：包含详细的类型提示和 Docstring。

## 第三步：编写采集器 (Extractors)
请在 `pipeline/extractors/` 下分别实现三个采集器：
1. **新闻 (news.py)**: 目标 `mining.com`。使用 `requests` + `BeautifulSoup`。处理分页，抓取最近新闻（标题、正文、时间）。需处理反爬（User-Agent池）。
2. **政策 (policy.py)**: 目标 `中国稀土集团` 或类似政府网站。编写通用的 `HTMLCleaner` 类去除 script/style 标签，提取主要文本。
3. **价格 (price.py)**: 目标 `LME` 或 `SHFE`。模拟 API 请求或解析表格。将价格数据转化为文本描述存入 `content`（例如："2026-07-01 铜价上涨至..."）。

## 第四步：ETL 主流程
在 `pipeline/etl.py` 中编写主流程：
1. 调用上述采集器。
2. **去重**：基于 `title` + `publish_date` 的哈希值去重。
3. **向量化**：使用 `sentence-transformers` 生成 embedding。
4. **入库**：存入 ChromaDB（本地持久化）。
5. 使用 `tqdm` 显示处理进度。

## 第五步：API 服务开发
在 `serve/main.py` 中搭建 FastAPI：
1. 启动时加载 ChromaDB。
2. 实现 `POST /query` 接口：接收 `{"question": "..."}` -> 向量化 -> 检索 Top 5 -> 返回结果。
3. 返回字段：`title`, `source_type`, `snippet`, `score`。
4. 添加 CORS 中间件。

## 第六步：评估与测试
1. 在 `eval/ground_truth.json` 生成 20 条模拟测试问答对（包含 `expected_keywords`）。
2. 编写 `eval/run_eval.py`：调用本地接口，计算 Recall@5，打印分数。

## 第七步：文档与交付
1. 生成 `DATA_NOTES.md`：数据库 Schema、去重逻辑、字段映射。
2. 生成 `README.md`：项目介绍、依赖安装、运行指令（ETL, Server, Eval）。
